TypeScript Project Structures and Development Models

This document outlines common project structures and architectural models used in TypeScript development. The examples below include configurations, directory structures, and best practices.

1. Basic Project Directory Structure

For small to medium-sized projects, the following structure is commonly used:

project-root/
├── src/                 # Source code
│   ├── components/      # UI components and modules
│   ├── services/        # API calls and business logic
│   ├── models/          # Types and interfaces
│   ├── utils/           # Utility functions
│   ├── main.ts          # Entry point
│   └── app.ts           # Application bootstrap file
├── dist/                # Compiled JavaScript files
├── node_modules/        # Dependencies
├── test/                # Test files
├── tsconfig.json        # TypeScript compiler settings
├── package.json         # Dependency and script definitions
└── README.md            # Project documentation
Note: The src directory contains the main application code, while the dist directory holds the compiled output.

2. Advanced Project Directory Structure

For larger and enterprise-level projects, a modular directory structure is recommended:

project-root/
├── src/
│   ├── modules/            # Modular structure
│   │   ├── user/           # User module
│   │   │   ├── controllers/
│   │   │   ├── models/
│   │   │   ├── services/
│   │   │   └── routes/
│   │   ├── product/        # Product module
│   │   │   ├── controllers/
│   │   │   ├── models/
│   │   │   ├── services/
│   │   │   └── routes/
│   ├── config/             # Configuration files
│   ├── shared/             # Shared utilities and types
│   ├── middleware/         # Middleware logic
│   ├── main.ts             # Entry point
│   └── app.ts              # Application bootstrap file
├── dist/                   # Compiled JavaScript files
├── public/                 # Static files
├── test/                   # Test files
│   ├── unit/               # Unit tests
│   ├── integration/        # Integration tests
│   └── e2e/                # End-to-end tests
├── docs/                   # Project documentation
├── tsconfig.json           # TypeScript compiler settings
├── package.json            # Dependency and script definitions
└── README.md               # Project documentation

3. TypeScript Compiler Workflow

The TypeScript compilation process involves several steps:

  1. Write source code in .ts files.
  2. Compile the code using tsc (TypeScript Compiler).
  3. Generated JavaScript files are placed in the dist directory.

Sample tsconfig.json File

{
    "compilerOptions": {
      "target": "ES6",
      "module": "CommonJS",
      "outDir": "./dist",
      "rootDir": "./src",
      "strict": true,
      "esModuleInterop": true,
      "forceConsistentCasingInFileNames": true
    },
    "include": ["src"],
    "exclude": ["node_modules", "dist"]
}

4. Architectural Models for Large Projects

1. MVC (Model-View-Controller)

Ideal for separating the data, UI, and business logic layers.

project-root/
├── src/
│   ├── models/            # Data models
│   ├── views/             # User interface
│   └── controllers/       # Business logic

2. Modular Architecture

Each feature is a self-contained module.

project-root/
├── src/
│   ├── user/
│   ├── product/
│   ├── order/

3. Monorepo

Managing multiple projects in a single repository.

project-root/
├── packages/
│   ├── app1/
│   ├── app2/
│   └── shared/

4. Hexagonal Architecture

Focuses on isolating the core business logic from external systems.

5. Event-Driven Architecture

Real-time communication between modules using event emitters.
